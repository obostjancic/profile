Underrated: Cloudflare tunnels

Nov 3, 2025

If you wanted to expose a node server running on port 3000 on a raspberry pi you would have to to a domain foo.bar that you own you would have to:

  1. Set the static IP address of the raspberry pi and configure it to use the static IP address
  2. Configure the router in which the Raspberry Pi is connected to forward external requests on a chosen port (e.g. 80 or 443) to the Raspberry Pi’s internal IP and port 3000 (port forwarding)
  3. Reserve or create a DNS A record for foo.bar (or a subdomain like pi.foo.bar) and point it to your router’s public IP address
  4. If your ISP assigns a dynamic public IP, set up Dynamic DNS (DDNS) so the domain always resolves to the current public IP, or obtain a static public IP from your ISP
  5. Obtain and install an SSL/TLS certificate (for example via Let’s Encrypt) for foo.bar so HTTPS connections are secure, and configure your server or a reverse proxy (nginx, Caddy) on the Raspberry Pi to use it

    6. or you could set up a cloudflare tunnel by running 

    brew install cloudflared
    cloudflared tunnel create foobar
    cloudflared tunnel route dns foobar foo.bar

    and that's it

TIL: Raspberry Pi 4 can't connect to WPA3 networks

Oct 24, 2025

After a few hours of running in circles with

nmcli connection add type wifi con-name foobar ssid foobar

then

nmcli connection modify foobar wifi-sec.key-mgmt sae
nmcli connection modify foobar wifi-sec.psk 12345678
nmcli connection up foobar

and endless

sudo systemctl restart NetworkManager

I have figured out that the Raspberry Pi 4 is not compatible with WPA3 networks. Ordered a compatible USB WiFi adapter and that worked like a charm.
